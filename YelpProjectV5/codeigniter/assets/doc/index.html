Yelp dataset a.k.a LocalSpot

Description

Information about businesses has a big impact on people now days because information is the power and when you have precise data you can control and make powerfull statistics about the future.

LocalSpot is a web application made to deliver users a representations of the yelp dataset. Once an application user logs in he can view statistics of his own prefered bussineses. There is also a posibility to have guest users, but they will not have the feature of saving and track the statistics of different companies.

The statistics are displayed via charts in order to give the user proper information about different businesses and helps the user to make a prevision about the future of different business based on the information provided by the yelp dataset.

Functionalities offered through the interface

- Login: The user will be able to login in the application in order to track different businesses.

- Logout: The user will be able to logout from the application whenever he wants to.

- Charts: The user will be able to see the yelp dataset throught graphical charts.

- Search: This option will allow the user to search throught the database for different businesses.

Shareholders & Interests

User: the possibility to access all functions of the yelp web application without any problem.

Interface: to provide an intuitive user interaction with the application and to achieve efficient communication with the futures.

Integration: to receive the proper data from the yelp dataset.

Formatting: the possibility to provide a well structured information via charts.

Actors & Objectives

User: provides the possibility to view and store the preferred businesses.

Interface: provides a way of communication between the application functionality and the user.

Integration: provides a well structured data from the yelp dataset.

Formatting: provides an graphic representation of the data via charts.

Scenarios of use

The following scenarios describe how the application is used by the client:

5.1 The user creates an account

5.1.1 Pressing the Sign Up button

5.1.1.1 Objective / Context

The user uses the sign up button to make an account.

5.1.1.2 Scenario / Steps

1. Pressing the sign up button will open a new page with text inputs.

2. On the sign up page the user will have to put his desired information.

3. Pressing the Sign Up button, will cause the application to create a user and register him into the database.

5.1.1.3 Extensions

If the user will not introduce a correct e-mail address he will be prompt with an error message telling him that the e-mail address entered is not valid.

5.1.2 Pressing the Log in button

5.1.2.1 Objective / Context

The user may have the possibility to log out from the application.

5.1.2.2 Scenario / Steps

1. Pressing the log out button will cause the application to log out the user

2. The application will send the user to the main page of the application.

5.2 Searching for data

5.2.1 Objective / Context

The user wants to search for data in yelp dataset.

5.2.2 Scenario / Steps

The user writes in the searching box the desired data and press enter.

5.2.3 Extensions

In case the user is typing a random and difficult to decipher word then the search system will not display such information since it does not exist in the database.
